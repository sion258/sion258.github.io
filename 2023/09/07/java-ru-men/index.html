Jvav Java入门

一. Java的特点

Java除了简单，面向对象的编程特点之外，其最大的特点就是平台无关性。

1. 平台无关性

类似C，C++等语言只能对特定的处理器（Central Processing Units，CPU）芯片进行编

译，当设变发生改变或者更换芯片之后就不能保证程序的正常进行，需要修改程序以适应编

译。

Java语言可以在操作系统上提供一个Java运行环境（Java Runtime Environment，

JRE），该运行环境由Java虚拟机（Java Virtual Machine，JVM），类库和一些核心软件

组成。

因此，只要平台提供Java运行环境，Java编写软件在上也能运行。

平台与机器指令

编程语言写出来的程序，都需要操作系统（Operating System，OS）和处理器完成，平台

就是指OS和CPU所构成。

每个平台都会形成自己独特的机器指令，例如，00001111可能指的是加法指令，在另外的

平台却是用10101010表示加法。OS和CPU不同都有可能导致。

Java虚拟机和字节码

JRM的核心式字节码指令，字节码指令不能被平台直接识别，只对于不同平台提供JRM直接

识别执行。JRM再将字节码翻译成对应平台机器码，交给本地平台运行。

2. 多线程

Java的特点之一就是内置对多线程的支持。多线程允许同时完成多个任务。实际上多线

程使人产生多个任务在同时执行的错觉，因为目前的计算机的处理器在同一时刻只能执行一

个线程，但处理器可以在不同的线程之间快速地切换，由于处理器速度非常快，远远超过了

人接收信息的速度，所以给人的感觉好像多个任务在同时执行。C++没有内置的多线程机制，

因此必须调用操作系统的多线程功能来进行多线程程序的设计。

3. 动态

Java程序的基本组成单元就是类，有些类是自己编写的，有些是从

类库中引入的，而类又是运行时动态装载的，这就使得Java可以在

分布环境中动态地维护程序及类库。C/C++编译时就将函数库或类

库中被使用的函数、类同时生成机器码，那么每当其类库升级之

后，如果C/C++程序想具有新类库提供的功能，程序就必须重新修

改、编译。

Java程序的开发步骤

  • 编写源文件

  • 编译源文件

  • 运行程序

    具体的过程是：利用编译器编译源文件，产生字节码，使用解

    释器执行字节码运行程序。

简单的Java应用程序

public class Hello{
	pubilc static void main(String args[]){
		System.out.println("hello");
		System.out.println("nice to meet you");
		Student stu = new Student();
		stu.speak("We are student");
	}
}
class Student{
	public void speak(String s){
		System.out.println(s);
	}
}

在上面，我们编写了一个源文件，注意在编写文件时注意代码的整

体风格。

接下来，我们应保存源文件。

一个源文件只能有一个是public类，并且源文件名字与该public类

名相同，保存的扩展名是.java；如果没有public文件，那么源文件

名字只需和某个类名相同即可。例如，此源文件应命名为

Hello.java，注意区分大小写。

接着，我们可以使用Java编译器（javac.exe）对其进行编译。源文

件有几个类，编译源文件时就会生成几个字节码文件（.class），

这些字节码文件与源文件的存储目录相同。当源文件出现语法错误

时不生成字节码文件。可以在MS-DOS命令行窗口输入dir命令查看

当前目录下的文件和子目录名称。

最后运行程序，注意的是一个源文件必须有一个类包含**public **

**static void **main(String args[]) 方法，包含该方法的类称为主

类，例如例代码的主类是Hello类，因此，请确保main方法书写正

确，否则可能出现Exception in thread

“main”java.lang.NoClassFondError的错误提醒。

最后Java解释器（java.exe）解释字节码文件，需要注意的是Java

解释其执行的类名必须是不带扩展名的主类的名字。

Java能提供反编译，将字节码文件还原为源文件，方法是使用

javap.exe。例如：

C:\chapter1\>javap.Hello

也可以反向编译类库中的类：

C:\chapter1\>javap java.util.Date
